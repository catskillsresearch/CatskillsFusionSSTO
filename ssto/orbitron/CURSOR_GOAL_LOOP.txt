A) What "Plan" / planning-oriented chat is for
  In Cursor, switch the chat mode to **Plan** (mode picker next to the input,
  alongside Agent / Ask) when you want the model to **design and sequence work**
  with minimal or no repo edits: tradeoffs, file touch list, acceptance tests,
  and checkpoints. **Agent** mode is for applying edits and running commands.
  Flow: Plan to lock scope -> you approve -> Agent executes a slice -> you
  verify -> repeat.

B) Methodical knock-out pattern (your goals 1–5 style)
  1. Paste the numbered goals as the single source of truth for the session.
  2. Ask for one **checkpoint artifact** per goal (review doc, script, XML diff,
     screenshot checklist) so "done" is objective.
  3. After each goal: **you** run FG/JSBSim or WarpX once; if pass, say
     "checkpoint N OK" so the next goal does not inherit hidden debt.
  4. Keep goals in **one branch** or one PR series; avoid mixing unrelated refactors.
